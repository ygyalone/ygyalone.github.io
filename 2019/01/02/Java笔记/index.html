Java笔记

发表于 2019-01-02 | 更新于 2019-01-31

记录一些Java特性，便于复习。

this关键字

  1. 一般使用

    对象方法中代表对象自身。

  2. 内部类对象中的this

    可以明确指定外部类对象的引用，比如OuterClass.this。

static关键字

  1. 修饰属性

    static关键字修饰的属性是类属性。

  2. 修饰方法

    static关键字修饰的方法是类方法。

  3. 修饰内部类

    静态内部类。

final关键字

  1. 声明一个常量或者常量属性。
  2. 修饰方法，该方法能被继承，但是不能被重写。
  3. 修饰类，该类不能被继承。

代码块

代码块是用{}包起来的一块代码，有以下几种：

  1. 普通代码块

    方法中的代码块，起到作用域的作用。

  2. 构造代码块

    类定义中的代码块，每次生成对象的时候会调用。在构造方法之前调用。

  3. 静态代码块

    类定义中的代码块，有static关键字修饰。第一次使用类的时候会调用，只执行一次。

  4. 同步代码块

    方法中的代码块，有synchronized关键字修饰，同时需要指定一个对象锁。代码块中的代码是多线程安全的。

继承

  1. 不继承构造方法。
  2. 不继承static方法。
  3. 不继承final方法。
  4. 不继承private方法。

方法重载

方法名和返回值相同，参数列表不同。

方法重写

子类重写继承的方法，需要注意以下几点：

  1. 重写方法抛出的异常不能是父类方法抛出的异常的父类。
  2. 重写方法的访问级别不能低于父类方法的访问级别。
  3. private，static，final修饰的方法和构造方法不能被重写，因为这几个关键字修饰的方法是不能被继承的。

抽象类

抽象类作为类的抽象，有以下几个特点：

  1. 抽象类不能实例化。
  2. 抽象方法必须定义在抽象类中。
  3. 抽象类不一定含有抽象方法。

接口

  1. 接口中可以定义静态常量和对象方法。

  2. 静态常量默认是public static final修饰。

  3. 对象方法默认是public abstract修饰。

  4. 实现类必须实现协议方法，抽象类可以不实现协议方法。

  5. JDK1.8更新:

    • JDK1.8之后可以提供方法的默认实现，方法名前加default关键字。

    • JDK1.8之后可以再接口中定义静态方法。

内部类

内部类分为以下几种:

  1. 成员内部类

    会和外部类对象绑定，内部类对象可以访问外部类对象的成员和方法。

  1. 静态内部类

    不会和外部类对象绑定，内部类可以访问外部类的静态成员和静态方法。

  1. 局部内部类

    会和外部类对象绑定，内部类对象可以访问外部类对象的成员和方法。

  1. 匿名内部类

    会和外部类对象绑定，内部类对象可以访问外部类对象的成员和方法。

为什么需要使用内部类？

  1. 可以实现多继承。
  2. 可以更好地隐藏内部实现。比如遵循某个协议，但是并不想让外部知道时。

需要注意的地方：

成员内部类，局部内部类，匿名内部类生成的对象会持有外部类对象，某些情况会导致内存泄漏的问题，所以一般最好使用静态内部类，静态内部类对象可以通过弱引用的方式持有外部内对象，这样就不会有内存泄漏的问题。

包与访问修饰符

package关键字要在java文件的第一行，指定这个文件属于哪个包中。

访问修饰符 同一个类 同一个包 不同包子类 不同包非子类

public Y Y Y Y

异常

为什么使用异常：

  • 可以统一使用异常来处理业务流程的错误
  • 可以更好的编写错误处理代码？
  • 更好地管理错误处理代码？

  1. 异常继承结构：

    Throwable是异常的基类，直接子类有Error和Exception。

    Error是严重的问题，可能是系统级别的错误。程序一般不用处理此类错误。

    Exception是应用程序可以处理的异常情况，其中又分为受检异常和非受检异常。受检异常是Exception类型或其子类型（非RuntimeException类型），在编译期要求强制处理这类异常，或者往上抛出。非受检异常是RuntimeException类型或其子类型，编译期不要求强制处理这类异常，运行期如果抛出这类异常而没有处理，会导致程序崩溃。

  2. try，catch

    使用try，catch来处理会方法抛出的异常。如果方法抛出异常，会按照catch代码块编写顺序寻找一个匹配的代码块并执行。

  3. finally

    • 不管有没有异常，finally中的代码都会执行。
    • 如果在try代码块中return，finally会在return前执行。

  4. throw，throws

    有受检异常抛出的方法在声明中需要添加throws关键字和抛出异常的类型，方法内部通过throw关键字将异常抛出。

  5. 自定义异常

    自定义异常可以继承Exception或者RuntimeException。至于继承哪一个，可以根据业务需求来决定，一般RuntimeException使用较多。

注解

注解分为以下两种：

  • 元注解，补充说明其它注解。
  • 自定义注解，补充说明代码。

元注解有以下几种：

  1. @Retention

    指定注解的保留策略

枚举值 保留策略
RetentionPolicy.SOURCE 作用于源代码中，不会被编译到Class文件
RetentionPolicy.CLASS 注解将被记录在Class文件中，程序运行时不会保留
RetentionPolicy.RUNTIME 注解将被记录在Class文件中，并且在程序运行时也会保留

  1. @Target

    指定注解所修饰的作用域

枚举值 作用域
ElementType.TYPE 类型
ElementType.FIELD 字段
ElementType.METHOD 方法
ElementType.PARAMETER 方法参数
ElementType.CONSTRUCTOR 构造方法
ElementType.LOCAL_VARIABLE 局部变量
ElementType.ANNOTATION_TYPE 注解
ElementType.PACKAGE 包
ElementType.TYPE_PARAMETER 类型参数

  1. @Inherited

    指定注解被子类继承

  2. @Documented

    指定注解包含在javadoc中

Java已经提供了几个内建注解：

  1. @Override 修饰方法，表示该方法是重写父类的方法。

  2. @Deprecated 修饰方法，表示该方法已经过期，不建议使用。

  3. @SuppressWarnings 消除警告。

自定义注解的使用场景：

自定义注解的保留策略一般指定为RetentionPolicy.RUNTIME。这样就可以在运行时通过反射拿到注解，从而执行一些其它操作。比如单元测试框架JUnit，在运行时通过反射获取有@Test注解标记的方法，这些方法将被作为测试方法执行。

其他

  1. 属性可以设置默认值。

  2. 方法入参不可以设置默认值。

  3. 没有闭包(或者说函数)类型，回调只能通过[匿名内部类]对象来实现。不同于C，Objective-C和Swift。有点蛋疼…

    补充:JDK1.8之后提供了 lambda 表达式，类似匿名函数。当接口中只有一个抽象对象方法时，可以使用 lambda 表达式代替[匿名内部类]对象来使用。虽然有lambda表达式，但Java还是没有提供函数这种类型，为了使用函数这种类型，必须要定义出一个接口出来的使用方式，感觉很奇葩。
